把下面整段复制到 **Figma Make** 作为生成提示。要求：**延续现有 App 视觉**，不要换品牌风格。

## 设计系统（必须与现有 BigChef 移动端一致）

参考当前 RN 应用视觉基准（从代码提取）：

- **页面背景**：`#fafafa`
- **主色 / 强调色**：`#ff6b35`（Tab 选中、主按钮、进度指示）
- **主文字**：`#13233a`
- **次级文字**：`#607086`、`#757575`
- **卡片**：白底 `#fff`，圆角 **16**，细描边 `#e8edf3`，轻阴影（`shadowOpacity ~0.06`，`shadowRadius ~10`，`shadowOffset y=4`）
- **Tab 栏**：白底，顶部分割线 `rgba(0,0,0,0.08)`，高度约 **66**；选中 Tab 图标容器浅橙底 `rgba(255,107,53,0.1)`
- **排版**：标题约 **17–18** semibold；辅助说明 **12–13**；列表舒适行高
- **图标**：可继续用 emoji 风格（🍳 🔍 👤）或换成同气质的线性图标，但**色调与圆角体系不变**

整体气质：**干净、温暖、生活感**，避免酷炫深色科技风或过度插画。

## 产品上下文

BigChef 从「做饭工具」升级为「关系驱动的做饭社区」：

1. **社区**：用户用手机相册**上传做菜短视频**（主路径），图文可选；可关联菜谱 `dishId`；Feed 支持「最新 / 热门」。
2. **家庭空间**（情侣或合租饭搭子）：**弱确认**协作周菜单；**心愿菜**；**常做菜统计**；**聚合采购清单**（合并食材）；支持单次/两阶段采购批次提示。
3. **临时聚餐**：发起人定人数、拉群、每人报菜、**一键生成菜单**；生成后配置**酒水偏好**（预设 + 自由输入）。

## 需要你输出的界面（移动端竖屏，375×812 基准）

请输出一套 **可开发对齐的高保真** UI（Auto Layout，组件化），包含以下屏幕与状态：

### A. 底部导航扩展

在现有「推荐 / 查菜 / 我的」之外，增加第 4 个 Tab **「社区」**（或把「社区」放在推荐页顶部入口，二选一，请给出你推荐的信息架构并说明理由）。

### B. 社区

1. **社区首页 Feed**  
   - 顶部：分段控件 **最新 / 热门**  
   - 卡片：封面图 16:9 或竖版缩略、标题、作者、点赞/评论数、视频时长角标  
   - **处理中**状态：灰色占位 +「视频处理中」  
   - **失败**状态：提示 +「重试」

2. **发布页（视频主路径）**  
   - 主 CTA：**从相册选择视频**  
   - 次入口：外链视频（弱化）  
   - 字段：标题、正文、关联菜谱（搜索选择）、标签 chips  
   - 上传进度条、**取消**  
   - 合规提示小字：时长/大小限制（120s / 200MB）

3. **帖子详情**  
   - 视频播放器区域 + 标题/正文  
   - 点赞、收藏、评论列表  
   - 评论输入框  

### C. 家庭 / 情侣

1. **创建家庭**  
   - 家庭名称  
   - **关系类型**：情侣 / 合租同伴（单选，说明文案不同）  
   - 生成邀请码展示 + 分享按钮（样式即可）

2. **加入家庭**  
   - 展示家庭名称 + 关系类型 + **勾选确认**「已知并同意」  
   - 主按钮加入 / 次按钮取消

3. **家庭主页**  
   - 本周吃啥（周视图条或日历条）  
   - 入口：周菜单、采购清单、常做菜统计、成员

4. **周菜单页**  
   - 按天列出晚餐（MVP 仅晚餐）  
   - **心愿池**列表 + 拖入某天的示意（可用「分配到周一」操作表）  
   - 主按钮：**一键生成一周菜单（AI）**、次按钮：通知对方看一眼（弱提醒，非强制确认）  
   - 顶部说明：**可随时改菜单，不强制双方点确认**

5. **采购清单**  
   - 合并食材行（名称、数量、批次标签「第 1 波 / 第 2 波」）  
   - 勾选已买、**家里已有**排除  
   - 单次采购 / 两阶段采购切换  
   - 建议购买日（周末 / 周中）用轻量提示条

6. **常做菜统计**  
   - Top 列表 +「距上次做」  
   - 行内按钮：**加入下周**

### D. 临时聚餐

1. **创建聚餐**：人数、标题、分享链接/口令样式  
2. **聚餐协作页**：成员列表、每人报菜（可选菜谱或短文本）  
3. **生成结果页**：菜单列表 + **酒水偏好**（啤酒/红酒/白酒/软饮/不饮酒 等多选 + 自由输入）  
4. 开关：**酒水是否进入采购清单**（默认关，用说明文案）

## 交互与空状态

- 所有列表：骨架屏 / 空状态插画 + 一句「过日子式」文案（轻松、不鸡汤）  
- 错误：顶部轻横幅或行内提示，色调用红色系但保持柔和（参考现有 `#fee2e2` / `#b91c1c`）

## 交付格式（给 Figma Make 的执行要求）

- 使用 **Auto Layout**，命名清晰（`Community/FeedCard`、`Family/WeekMenu`…）  
- 给出 **Light mode** 即可；标注关键间距（8pt 栅格）  
- 组件：**Primary Button**、**Secondary Button**、**Chip**、**List Row**、**Section Card** 与现有 `SectionCard` 视觉一致  
- 导出时附 **1 页 Overview**：把信息架构用小地图/箭头标出用户主路径：「上传视频 → 帖子 → 关联菜谱」「心愿 → 一键周菜单 → 采购清单」「聚餐报菜 → 生成 → 酒水」
